# Code Generation Scripts

This directory contains code generation scripts for the desktop app.

## generate_crud.go

Generates Master Data CRUD methods from YAML configuration.

### Usage

```bash
# Run manually
go run scripts/generate_crud.go

# Or use go generate (recommended)
go generate ./...
```

### How It Works

1. Reads `masterdata_entities.yaml` in project root
2. For each entity, generates:
   - `GetXPaginated(requestJSON string) (string, error)`
   - `SearchX(searchRequestJSON string) (string, error)`
3. Outputs to `masterdata_generated.go`

### Adding New Entity

Edit `../masterdata_entities.yaml`:

```yaml
entities:
  - name: NewEntity      # PascalCase name
    plural: newentities  # lowercase plural for error messages
    service_method: NewEntity  # Method name on PKSMasterService
```

Then run:
```bash
go generate ./...
```

### Output

Generates `masterdata_generated.go` with:
- Header comment: `// Code generated by go generate; DO NOT EDIT.`
- All required imports
- 2 methods per entity
- ~50 lines of code per entity

### Example

Input (YAML):
```yaml
- name: Produk
  plural: products
  service_method: Produk
```

Output (Go):
```go
func (a *App) GetProdukPaginated(requestJSON string) (string, error) {
    // ... implementation
}

func (a *App) SearchProduk(searchRequestJSON string) (string, error) {
    // ... implementation
}
```

## Dependencies

- `gopkg.in/yaml.v3` - YAML parsing

Install with:
```bash
go get gopkg.in/yaml.v3
```

## Troubleshooting

**Error: cannot find masterdata_entities.yaml**
- Make sure you're running from project root
- Check file exists: `ls masterdata_entities.yaml`

**Error: output file already exists**
- Delete `masterdata_generated.go` and re-run
- Or: generator will overwrite automatically

**Generated code doesn't compile**
- Check YAML syntax
- Verify `service_method` matches actual service method name
- Check imports in template match your project structure
